OK。**DBなしで始めて、あとからDBに置き換える流れ**を、筋トレアプリの `Exercise` で説明します。

全体像はこれです。

```text
1. 扱うデータを決める
↓
2. 仮データをリスト・辞書で作る
↓
3. スキーマを書く
↓
4. APIルーターを書く
↓
5. リスト・辞書でCRUD処理を書く
↓
6. 動作確認する
↓
7. DBモデルを書く
↓
8. DBと接続する
↓
9. リスト保存をDB保存に置き換える
↓
10. もう一度動作確認する
```

---

# 1. まず扱うデータを決める

最初に考えるのは、

**このアプリで何のデータを扱うのか？**

です。

筋トレアプリなら、例えばこうです。

```text
Exercise = 種目
BodyPart = 部位
ExerciseType = 種目タイプ
Level = レベル
User = ユーザー
WorkoutLog = トレーニング記録
```

最初から全部作ると大変なので、まずは `Exercise` だけでOKです。

`Exercise` に必要な情報を考えます。

```text
id
name
description
target_body_part
exercise_type
level
```

たとえばこういうデータです。

```python
{
    "id": 1,
    "name": "デッドリフト",
    "description": "背中と脚を鍛える種目",
    "target_body_part": "背中",
    "exercise_type": "筋トレ",
    "level": "中級"
}
```

この段階ではまだDBもスキーマも不要です。

まずは、

**このアプリではこういうデータを扱うんだな**

と決めるだけです。

# 3. スキーマを書く

次にスキーマを書きます。

スキーマは、

```text
外から来るJSONのルール
```

です。

まずは作成用のスキーマを作ります。

```python
from pydantic import BaseModel
from typing import Optional


class ExerciseCreate(BaseModel):
    name: str
    description: Optional[str] = None
    target_body_part: str
    exercise_type: str
    level: str
```

これは、

```json
{
  "name": "ベンチプレス",
  "description": "胸を鍛える種目",
  "target_body_part": "胸",
  "exercise_type": "筋トレ",
  "level": "初級"
}
```

みたいなJSONを受け取るためのルールです。

ここでは `id` は入れません。

なぜなら `id` はユーザーに送らせるものではなく、サーバー側で作るものだからです。

---

次に、返す用のスキーマも作れます。

```python
class ExerciseOut(BaseModel):
    id: int
    name: str
    description: Optional[str] = None
    target_body_part: str
    exercise_type: str
    level: str
```

作成用と返す用の違いはこうです。

```text
ExerciseCreate
→ ユーザーから受け取る形
→ id はいらない

ExerciseOut
→ ユーザーに返す形
→ id も含める
```

---

# 4. APIルーターを書く

次にAPIを作ります。

FastAPIなら、まずこんな感じです。

```python
from fastapi import FastAPI

app = FastAPI()
```

そして、仮データを置きます。

```python
exercises = [
    {
        "id": 1,
        "name": "デッドリフト",
        "description": "背中と脚を鍛える種目",
        "target_body_part": "背中",
        "exercise_type": "筋トレ",
        "level": "中級",
    }
]
```

まずは一覧取得APIを作ります。

```python
@app.get("/exercises")
def get_exercises():
    return exercises
```

これで `/exercises` にアクセスすると、種目一覧が返ります。

---

次に、1件だけ取得するAPI。

```python
@app.get("/exercises/{exercise_id}")
def get_exercise(exercise_id: int):
    for exercise in exercises:
        if exercise["id"] == exercise_id:
            return exercise

    return {"message": "Exercise not found"}
```

これで、

```text
/exercises/1
```

にアクセスすると、id が 1 の種目を返します。

# 5. リスト・辞書でCRUD処理を書く

CRUDはこれです。

```text
Create = 作成
Read   = 取得
Update = 更新
Delete = 削除
```

まず作成です。

```python
@app.post("/exercises")
def create_exercise(exercise: ExerciseCreate):
    new_exercise = {
        "id": len(exercises) + 1,
        "name": exercise.name,
        "description": exercise.description,
        "target_body_part": exercise.target_body_part,
        "exercise_type": exercise.exercise_type,
        "level": exercise.level,
    }

    exercises.append(new_exercise)

    return new_exercise
```

流れはこうです。

```text
JSONが送られてくる
↓
ExerciseCreate がチェックする
↓
new_exercise を作る
↓
exercises に追加する
↓
追加したデータを返す
```

---

更新はこうです。

```python
class ExerciseUpdate(BaseModel):
    name: Optional[str] = None
    description: Optional[str] = None
    target_body_part: Optional[str] = None
    exercise_type: Optional[str] = None
    level: Optional[str] = None
```

更新用スキーマでは全部 `Optional` にします。

理由は、一部だけ更新したいからです。

例えば、

```json
{
  "level": "上級"
}
```

だけ送って更新できるようにします。

APIはこうです。

```python
@app.patch("/exercises/{exercise_id}")
def update_exercise(exercise_id: int, exercise_update: ExerciseUpdate):
    for exercise in exercises:
        if exercise["id"] == exercise_id:
            if exercise_update.name is not None:
                exercise["name"] = exercise_update.name

            if exercise_update.description is not None:
                exercise["description"] = exercise_update.description

            if exercise_update.target_body_part is not None:
                exercise["target_body_part"] = exercise_update.target_body_part

            if exercise_update.exercise_type is not None:
                exercise["exercise_type"] = exercise_update.exercise_type

            if exercise_update.level is not None:
                exercise["level"] = exercise_update.level

            return exercise

    return {"message": "Exercise not found"}
```

---

削除はこうです。

```python
@app.delete("/exercises/{exercise_id}")
def delete_exercise(exercise_id: int):
    for exercise in exercises:
        if exercise["id"] == exercise_id:
            exercises.remove(exercise)
            return {"message": "Exercise deleted"}

    return {"message": "Exercise not found"}
```

これで最低限のCRUDができます。

# 7. DBモデルを書く

次にDBを入れます。

ここで出てくるのがDBモデルです。

スキーマは注文用紙。

DBモデルは倉庫構造です。

SQLAlchemyなら、例えばこうです。

```python
from sqlalchemy import Column, Integer, String
from database import Base


class Exercise(Base):
    __tablename__ = "exercises"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String, nullable=False)
    description = Column(String, nullable=True)
    target_body_part = Column(String, nullable=False)
    exercise_type = Column(String, nullable=False)
    level = Column(String, nullable=False)
```

これは、

```text
exercises テーブルを作る
その中に
- id
- name
- description
- target_body_part
- exercise_type
- level
という列を作る
```

という意味です。

---

スキーマとDBモデルを比べるとこうです。

スキーマ。

```python
class ExerciseCreate(BaseModel):
    name: str
    description: Optional[str] = None
    target_body_part: str
    exercise_type: str
    level: str
```

DBモデル。

```python
class Exercise(Base):
    __tablename__ = "exercises"

    id = Column(Integer, primary_key=True)
    name = Column(String)
    description = Column(String)
    target_body_part = Column(String)
    exercise_type = Column(String)
    level = Column(String)
```

似ています。

でも役割が違います。

```text
スキーマ
→ APIで受け取る・返すデータの形

DBモデル
→ DBに保存するテーブルの形
```

---

# 8. DBと接続する

次に、アプリとDBをつなぎます。

例えば SQLite ならこうです。

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

DATABASE_URL = "sqlite:///./app.db"

engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False}
)

SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)

Base = declarative_base()
```

これは、

```text
SQLite の app.db に接続する
SQLAlchemy でDB操作できるようにする
```

という設定です。

さらに、FastAPIでDBセッションを使うためにこれを書きます。

```python
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

これはAPIの中でDBを使うための準備です。

# 9. リスト保存をDB保存に置き換える

ここが大事です。

今まではこうしていました。

```python
exercises.append(new_exercise)
```

つまり、リストに保存していました。

DBありにすると、こうなります。

```python
db.add(db_exercise)
db.commit()
db.refresh(db_exercise)
```

つまり、DBに保存します。

---

例えば作成APIはこう変わります。

DBなし版。

```python
@app.post("/exercises")
def create_exercise(exercise: ExerciseCreate):
    new_exercise = {
        "id": len(exercises) + 1,
        "name": exercise.name,
        "description": exercise.description,
        "target_body_part": exercise.target_body_part,
        "exercise_type": exercise.exercise_type,
        "level": exercise.level,
    }

    exercises.append(new_exercise)

    return new_exercise
```

DBあり版。

```python
from fastapi import Depends
from sqlalchemy.orm import Session


@app.post("/exercises")
def create_exercise(
    exercise: ExerciseCreate,
    db: Session = Depends(get_db)
):
    db_exercise = Exercise(
        name=exercise.name,
        description=exercise.description,
        target_body_part=exercise.target_body_part,
        exercise_type=exercise.exercise_type,
        level=exercise.level,
    )

    db.add(db_exercise)
    db.commit()
    db.refresh(db_exercise)

    return db_exercise
```

流れは同じです。

```text
JSONが来る
↓
スキーマでチェック
↓
Exercise DBモデルに詰め替える
↓
DBに保存
↓
保存したデータを返す
```

保存先が変わっただけです。

```text
前: Pythonのリスト
後: データベース
```

---

一覧取得も変わります。

DBなし版。

```python
@app.get("/exercises")
def get_exercises():
    return exercises
```

DBあり版。

```python
@app.get("/exercises")
def get_exercises(db: Session = Depends(get_db)):
    return db.query(Exercise).all()
```

---

1件取得も変わります。

DBなし版。

```python
@app.get("/exercises/{exercise_id}")
def get_exercise(exercise_id: int):
    for exercise in exercises:
        if exercise["id"] == exercise_id:
            return exercise

    return {"message": "Exercise not found"}
```

DBあり版。

```python
from fastapi import HTTPException


@app.get("/exercises/{exercise_id}")
def get_exercise(
    exercise_id: int,
    db: Session = Depends(get_db)
):
    exercise = db.query(Exercise).filter(Exercise.id == exercise_id).first()

    if exercise is None:
        raise HTTPException(status_code=404, detail="Exercise not found")

    return exercise
```

---

更新も変わります。

DBあり版。

```python
@app.patch("/exercises/{exercise_id}")
def update_exercise(
    exercise_id: int,
    exercise_update: ExerciseUpdate,
    db: Session = Depends(get_db)
):
    exercise = db.query(Exercise).filter(Exercise.id == exercise_id).first()

    if exercise is None:
        raise HTTPException(status_code=404, detail="Exercise not found")

    if exercise_update.name is not None:
        exercise.name = exercise_update.name

    if exercise_update.description is not None:
        exercise.description = exercise_update.description

    if exercise_update.target_body_part is not None:
        exercise.target_body_part = exercise_update.target_body_part

    if exercise_update.exercise_type is not None:
        exercise.exercise_type = exercise_update.exercise_type

    if exercise_update.level is not None:
        exercise.level = exercise_update.level

    db.commit()
    db.refresh(exercise)

    return exercise
```

---

削除も変わります。

```python
@app.delete("/exercises/{exercise_id}")
def delete_exercise(
    exercise_id: int,
    db: Session = Depends(get_db)
):
    exercise = db.query(Exercise).filter(Exercise.id == exercise_id).first()

    if exercise is None:
        raise HTTPException(status_code=404, detail="Exercise not found")

    db.delete(exercise)
    db.commit()

    return {"message": "Exercise deleted"}
```

# 10. もう一度動作確認する

最後に、DBありで動作確認します。

確認することは同じです。

```text
GET /exercises
POST /exercises
GET /exercises/{id}
PATCH /exercises/{id}
DELETE /exercises/{id}
```

ただし、DBありにしたら大きな違いがあります。

**サーバーを止めてもデータが残ります。**

DBなし。

```text
サーバー停止
↓
データ消える
```

DBあり。

```text
サーバー停止
↓
DBに保存されている
↓
再起動してもデータが残る
```

ここで初めて「本物のアプリっぽく」なります。

# まとめると

最初からDBを入れると、こういうものを一気に理解しないといけません。

```text
FastAPI
Pydantic
SQLAlchemy
DB接続
Session
CRUD
テーブル設計
リレーション
マイグレーション
```

かなり多いです。

だから最初は、

```text
リスト・辞書で仮データ
```

から始める方が理解しやすいです。

おすすめの考え方はこれです。

```text
最初:
リスト = 仮の倉庫

慣れたら:
DB = 本物の倉庫
```

流れとしては、

```text
JSONが来る
↓
スキーマがチェックする
↓
APIが受け取る
↓
最初はリストに保存する
↓
あとでDBモデルに詰め替えてDBに保存する
```

です。

一番大事なのはこれです。

```text
スキーマ = 注文用紙のルール
DBモデル = 倉庫の構造
リスト・辞書 = 練習用の仮倉庫
DB = 本物の倉庫
```

なので学習の順番としては、

```text
まず仮倉庫で流れを覚える
↓
その後、本物の倉庫に置き換える
```

が一番わかりやすいです。
